STEP 01

VPS 구입 + SSH 키 등록

Hostinger VPS를 구입하고 SSH 키를 등록해 비밀번호 없이 root 접속을 준비합니다.

01-1. VPS 플랜 선택

Hostinger hPanel → VPS → 새 VPS 주문을 클릭합니다. OS는 Ubuntu 22.04 LTS를 선택하세요. 이 스택의 메모리 footprint는 약 750 MB (Paperclip ~600 MB + Hermes ~100 MB + ttyd ~10 MB)이므로 KVM 2 (2 GB RAM) 이상 플랜을 권장합니다. 구입 완료 후 hPanel에서 VPS IP 주소를 확인합니다.

01-2. SSH 키 생성 (로컬 터미널) 
# 로컬 macOS / Linux 터미널에서 실행
ssh-keygen -t ed25519 -f ~/.ssh/hostinger -C "hostinger-vps"

# 공개키 내용 확인 (hPanel에 붙여넣을 값)
cat ~/.ssh/hostinger.pub
01-3. hPanel에서 SSH 키 등록

hPanel → VPS → 서버 선택 → SSH Keys 탭 → Add SSH Key를 클릭합니다. 위에서 출력한 공개키(ssh-ed25519 AAAA... 전체)를 붙여넣고 저장합니다. 이미 비밀번호 로그인이 활성화된 서버는 이 단계에서 SSH 키를 추가하면 이후 키 인증만 사용할 수 있습니다.

01-4. 첫 SSH 접속 확인 
# <VPS-IP>를 hPanel에서 확인한 실제 IP로 교체
ssh -i ~/.ssh/hostinger root@<VPS-IP>

# 접속 성공 시 출력 예시:
# Welcome to Ubuntu 22.04.4 LTS (GNU/Linux 5.15.0-101-generic x86_64)
# root@vps-abc:~#
01-5. 확인 기준

SSH 키 인증으로 root@<VPS-IP> 접속이 성공합니다. 비밀번호 입력 없이 바로 쉘 프롬프트가 나타나면 준비 완료입니다.

STEP 02

git clone + setup.sh (MODE 선택)

VPS에서 저장소를 클론하고 setup.sh를 실행해 .env를 자동 생성합니다. local 또는 tailscale 중 배포 모드를 선택합니다.

02-1. Docker 설치 + 저장소 클론 (VPS 쉘에서)

apt update && apt install -y git docker.io docker-compose-plugin

# systemd 서비스 등록 + 즉시 시작
systemctl enable --now docker

# 저장소 클론
git clone https://github.com/dandacompany/paperclip-hermes-codex-on-hostinger.git
cd paperclip-hermes-codex-on-hostinger
MODE동작 방식권장 시나리오

tailscaleTailscale 메시 + HTTPS 인증서 자동 발급인터넷 어디서나 .ts.net URL로 접속
02-2. local 모드로 setup.sh 실행

02-3. tailscale 모드로 setup.sh 실행

.env 파일이 생성되었고 ADMIN_PASSWORD가 32자 무작위 문자열로 채워져 있습니다. cat .env 출력에서 필수 변수 전부가 보이면 다음 단계로 진행합니다.

STEP 06

Codex OAuth 부트스트랩

컨테이너 로그에서 device URL을 찾아 브라우저로 1회 인증합니다. 이후 재인증은 불필요합니다.

06-1. 로그에서 Codex OAuth URL 찾기 
docker compose logs paperclip-hermes-codex | grep -i "codex"

# 예상 출력:
# [codex-oauth] No auth.json found — starting device auth
# [codex-oauth] Open this URL to authenticate:
# [codex-oauth] https://auth.openai.com/device?user_code=XXXX-XXXX
# [codex-oauth] Code: XXXX-XXXX (expires in 15 minutes)
06-2. device URL 형식

URL은 https://auth.openai.com/device?user_code=XXXX-XXXX 형식입니다. 로그에서 해당 줄을 복사해 브라우저에 붙여넣습니다. 코드 만료 시간(15분) 안에 인증을 완료해야 합니다. 시간이 지났다면 컨테이너를 재시작하면 새 URL이 발급됩니다: docker compose restart paperclip-hermes-codex

06-3. 브라우저 인증 절차 
# 1. 로그에서 복사한 URL을 새 탭에서 엽니다
#    예: https://auth.openai.com/device?user_code=XXXX-XXXX

# 2. ChatGPT / OpenAI 계정으로 로그인 후 "Allow" 클릭

# 3. 인증 완료 후 로그에서 확인
docker compose logs paperclip-hermes-codex | grep "OAuth completed"
# 예상 출력:
# [codex-oauth] Codex OAuth completed — auth.json saved
06-4. API 키 fallback

OAuth 대신 API 키를 사용하려면 .env에 OPENAI_API_KEY=sk-...를 추가하고 컨테이너를 재시작합니다. Codex CLI 0.122+ 버전은 이 값을 자동으로 auth.json 형식으로 변환합니다. OAuth와 API 키 중 하나만 있으면 충분합니다.

06-5. auth.json 생성 확인 
docker exec paperclip-hermes-codex-on-hostinger-paperclip-hermes-codex-1   ls -la /home/node/.codex/

# 예상 출력:
# -rw------- 1 node node 312 Jan 01 12:00 auth.json
06-6. 확인 기준

로그에 [codex-oauth] Codex OAuth completed가 출력됩니다. /home/node/.codex/auth.json이 생성되었고 크기가 0 이상입니다. 이 파일은 codex-auth named volume에 영구 저장되므로 컨테이너 재시작 후에도 재인증이 필요 없습니다.

STEP 07

첫 CEO 에이전트 생성 + Test now

Paperclip UI에서 Hermes Agent 어댑터를 사용하는 CEO 에이전트를 만들고 어댑터 환경을 검증합니다.

07-1. 에이전트 생성 절차 
# Paperclip UI (http://127.0.0.1:3100 또는 https://<fqdn>:3100)
# 1. 왼쪽 사이드바 → Agents → + New Agent 클릭
# 2. Name: CEO (또는 원하는 이름)
# 3. Adapter type: "Hermes Agent" 선택
# 4. Hermes URL: http://localhost:9119  (컨테이너 내부 주소)
# 5. Role / Persona: 에이전트 역할 입력 (예: "You are a CEO agent...")
# 6. Save 클릭
07-2. Hermes Agent 어댑터가 동작하는 이유

v1 단일 컨테이너는 hermes 바이너리를 /opt/hermes/.venv/bin/에 포함합니다. Paperclip의 어댑터 환경 체크(Adapter env check)는 같은 컨테이너 PATH에서 바이너리를 즉시 발견하므로 추가 설치 없이 통과됩니다. PATH 에러가 발생하면 컨테이너 로그에서 [supervisor] hermes dashboard가 정상 시작했는지 확인합니다.

07-3. Test now 실행 
# 에이전트 상세 페이지 → "Test now" 버튼 클릭
# 성공 시 표시:
Adapter env check: passed
07-4. 확인 기준

Test now 클릭 후 Adapter env check: passed 메시지가 표시됩니다. 오류가 나면 docker compose logs paperclip-hermes-codex | grep supervisor에서 Hermes Dashboard 프로세스 상태를 확인합니다.

10-3. 4개 volume의 역할

paperclip-data: Paperclip 워크스페이스 데이터 (에이전트, 태스크, 설정) | hermes-data: Hermes 세션, 스킬 패키지, API 키 | codex-auth: Codex OAuth 토큰 (auth.json) | tailscale-state: Tailscale 노드 ID + MagicDNS 인증서. 4개 모두 백업해야 완전 복원이 가능합니다. tailscale-state를 복원하면 동일 Tailscale 노드 ID로 재연결되어 FQDN이 유지됩니다.

10-4. 확인 기준

백업 후 docker compose down -v로 volume을 완전 삭제하고, up -d + restore 순서로 복원한 뒤 Paperclip 로그인이 가능하면 백업 완전성이 검증됩니다. Codex OAuth를 다시 하지 않고 Task 실행이 성공하면 codex-auth volume 복원도 정상입니다.
